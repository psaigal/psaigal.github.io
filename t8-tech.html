Recursion

August 9, 2015

People often joke that in order to understand recursion, you must first understand recursion.

John D. Cook

Recursion is a concept used in all programming languages. It is defined as a function calling on itself. Using recursion can allow your code to be more concise, simple, and understandable.

Simple Example

Let's start off by looking at a simple example that does not use recursion. The example below shows the function cookieJar, which takes a number as its parameter. In the for loop, the variable cookies is initially set to 0, and it is incremented by 1 each time the code block is executed. When the cookies variable is equal to the function's argument, there will be no more cookies left to add to the cookie jar. If the cookies variable is less than the function's number argument, the message "Keep adding more cookies" will be printed to the console.


var cookieJar = function(number){
  for (var cookies = 0; cookies<=number; cookies++){
    if (cookies==number){
    return console.log("There are no more cookies to add!");
    }
    console.log("Keep adding more cookies!");
  }
}


cookieJar(5)
=>  Keep adding more cookies!
    Keep adding more cookies!
    Keep adding more cookies!
    Keep adding more cookies!
    Keep adding more cookies!
    There are no more cookies to add!
Recursion Example

Below, you will see the same example utilizing recursion. The first if statement is the base case. When this line of code is executed, it will stop the recursion. Without this statement, the recursion will never stop, leading your computer to crash! Moving to the else statement, you will see that if the number argument is greater than 0, the message "Keep adding cookies!" will print to the console. This is known as the recursive case. Each time the code block is executed, the variable countDown is equal to the number-1. After that line, we see the function calling itself. This continues until the base case. There are no for or while loops used unlike the previous example.


var cookieJar = function(number){
    if (number==0) {
    return console.log("There are no more cookies to add!");
    }
    else {
    console.log("Keep adding more cookies!");
    var countDown = number-1;
    cookieJar(countDown);
    }
}



cookieJar(5)
=>  Keep adding more cookies!
    Keep adding more cookies!
    Keep adding more cookies!
    Keep adding more cookies!
    Keep adding more cookies!
    There are no more cookies to add!
When To Use Recursion

The memory set aside each time a function is called is known as a frame. In a recursive example, many functions are being called, and their frames are stored into a call stack, which is a LIFO data-type (Last In First Out). Since recursive functions can get really complex and deep, they often require a great deal of memory and can cause a program to run slow. For that reason, it is important to use recursion only when it is truly necessary.
