Version control is a system that allows a team to track, modify, and share different versions of a project. Individuals can make changes to their work that will be monitored. These changes are then submitted and recorded. Team members can constantly obtain the newest revision by requesting a pull. Different members of a team can make revisions to the same project at the same time without affecting each other. When the revisions are completed, they can merge their changes back to the central project. Git is a distrubuted version control system. This means every team member has their own repository which they can change without connecting to the server every time. All revisions can be pushed into the main repository.

Git has a sophisticated system of keeping track of changes. The "git status" command will allow you to obtain information about the current state of your files. It will designate files as untracked if they were not present in the last commit. The "git add" command will track a file, and it will be staged to be committed. If you make changes to a file that was tracked at one point, then git will send out a message saying the file has been tracked but it is not staged to be committed. Again, the "git add" command will allow it to be staged. You can commit the files using the "git commit" command. Every file that has been committed or is staged to be committed will have a unique SHA-1 hash, which is essentially a 40 hex character "stamp". This allows Git to keep track of all the changes that have ever been made to a specific file.

GitHub has a number of unique features. The social network aspect of the website allows users to follow one another and learn more about other user's contributions. The GitHub fork, pull request, and merge features allow users to easily and efficiently make changes to different open-source projects. This free platform makes it easy for novice programmers to learn and grow. In addition, users can easily store all of their code in one location.
