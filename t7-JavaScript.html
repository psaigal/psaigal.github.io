Ruby Hashes vs. JavaScript Objects

August 2, 2015

Javascript and Ruby are both powerful programming langauges. The two languages have many similarities and differences. Javascript objects are essentially a colection of properties. They are very similar to Ruby hashes.

Javascript Objects

Below, an object is assigned to the variable familyDog. The properties of the object are the variables name, age, furColor, and eyeColor. The values are shown after the colon. While the key values can be a Javascript data type such as a string, integer, or boolean, the properties are always variables. Each property is separated by a comma, except for the final one. To access an objects value, use object.property. To assign a new value to an object's property, you simply set the object.property to equal the new value. To delete an object's property(age in this case), use delete. You can see that the property and its value are removed from the object.

Ruby Hashes

Ruby hashes allows you to store information using key value pairs. Unlike Javascript, the keys in Ruby hashes do not have to be variables. In addition, the key value pairs are not separated by a colon. Fetching and changing values are done in a similar manner to Javascript objects. To delete a property, you use the delete method.
